‚öôÔ∏è Core Architecture Overview

Beto Factory Core Edition retains the essential five-step architecture from concept to deployment.

  1. HTML Input: Raw design templates.
  2. Canonical Model: Structured JSON layout representation.
  3. Platform Adapter: Converts canonical JSON ‚Üí platform-specific templates.
  4. Output Cleaner: Validates, optimizes, and minimizes export.
  5. ZIP Export & Deploy: Generates beto-theme.zip ready for upload.

üìò Canonical Model Schema Definition

{
  "$schema": "https://beto.factory/schema/canonical.json",
  "layout": {
    "header": "default",
    "footer": "default"
  },
  "components": {
    "hero": {"type": "banner", "props": {"title": "Welcome"}},
    "product-grid": {"type": "grid"}
  },
  "assets": {"images": ["logo.png", "hero.jpg"]}
}

This defines the universal structure that every adapter reads to generate platform-specific templates.

üß© Adapter Interface Specification

interface Adapter {
  name: string;
  input: CanonicalModel;
  mapComponent(name: string, template: string): void;
  generateConfig(): ThemeConfig;
  export(outputDir: string): Promise;
}

This contract ensures all adapters share a consistent implementation. Future platforms can plug in seamlessly without rewriting the core logic.
